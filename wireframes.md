# Tensionote 页面级线框说明文档

## 1. 文档目标

本文档用于定义血压管理 App V1 的页面结构、模块布局、页面跳转关系和关键交互状态。

本文档不是高保真视觉稿，而是用于指导：

- 产品确认页面结构
- 设计师输出 UI 视觉稿
- 开发理解页面组成和交互逻辑

本文档与 [requirements_prompt.md](D:\code\BloodPressionMaster\requirements_prompt.md)、[PRD.md](D:\code\BloodPressionMaster\PRD.md) 和 [native_technical_architecture.md](D:\code\BloodPressionMaster\native_technical_architecture.md) 保持一致。

## 2. 导航结构

V1 建议采用底部 Tab 导航，共 4 个一级入口：

1. 首页
2. 历史
3. 提醒
4. 设置

以下页面通过二级跳转进入：

- 新增记录页
- 编辑记录页
- 记录详情页
- 趋势详情页
- 报告页
- 隐私政策页
- 用户协议页
- 免责声明页

## 3. 首页

### 3.1 页面目标

首页是 V1 最重要页面，承担以下任务：

- 让用户直接快速录入最基础血压数据
- 展示最近 2 周血压趋势
- 让用户直接看懂哪一项偏高或偏低

首页只保留两个核心区域：

1. 快速录入
2. 最近 2 周历史曲线

其余能力例如报告、提醒、历史列表、合规文档，均由其他页面承载，不放在首页。

### 3.2 页面结构

建议自上而下布局如下：

1. 顶部极简导航区
- 页面标题
- 可选仅保留一个次级入口，例如“更多录入”

2. 快速录入区
- 直接输入最基础值
- 收缩压输入框
- 舒张压输入框
- 心率输入框
- 立即保存按钮
- 进入详细录入页入口

3. 趋势图区
- 标题：最近 2 周趋势
- 收缩压/舒张压曲线
- 正常范围带
- 异常点高亮
- 查看全部趋势入口
- 当前选中点的直接解释文案
- 文案必须明确说明是收缩压偏高、舒张压偏高、两者都偏高，还是处于常见正常范围

### 3.3 页面草图

```text
+--------------------------------------------------+
| 血压记录                                  详细录入 |
+--------------------------------------------------+
| 快速录入                                          |
| 收缩压 [    ]   舒张压 [    ]   心率 [    ]       |
| [ 立即保存 ]                        [详细录入]    |
+--------------------------------------------------+
| 最近 2 周趋势                                    |
| [ 收缩压/舒张压曲线 + 正常范围带 + 异常点 ]      |
|                                [查看全部趋势]    |
+--------------------------------------------------+
| 03/28 20:10：收缩压偏高，舒张压也偏高            |
+--------------------------------------------------+
```

### 3.4 关键交互

- 在首页输入基础数值后点击立即保存：直接创建记录并刷新曲线
- 点击详细录入：进入新增记录页
- 点击趋势图：进入趋势详情页
- 点击曲线上的点：在图下方显示该点的直接解释文案
- 点击查看全部趋势：进入趋势详情页

### 3.5 关键状态

无数据状态：

- 趋势图区域显示占位说明
- 主文案强调“先录入一条血压记录”
- 不显示其它摘要卡片

异常状态：

- 趋势图异常点使用高对比色
- 曲线下方直接显示解释，例如“收缩压偏高”或“舒张压偏高”
- 不使用需要用户自己判断含义的抽象标签作为唯一表达

## 4. 新增记录页

### 4.1 页面目标

让用户以最少步骤完成一次血压录入。

### 4.2 页面结构

1. 顶部导航栏
- 返回按钮
- 标题：新增记录

2. 输入区域
- 收缩压输入框
- 舒张压输入框
- 心率输入框
- 测量时间选择器

3. 场景标签区域
- 晨起
- 午后
- 晚间
- 饭后
- 运动后
- 服药后
- 不适

4. 备注区域
- 可选多行输入框

5. 底部主按钮
- 保存记录

### 4.3 页面草图

```text
+--------------------------------------------------+
| 返回                  新增记录                   |
+--------------------------------------------------+
| 收缩压 [      ] mmHg                             |
| 舒张压 [      ] mmHg                             |
| 心率   [      ] bpm                              |
| 测量时间 [ 2026-03-29 08:30 ]                    |
+--------------------------------------------------+
| 场景标签                                         |
| [晨起] [午后] [晚间] [饭后]                      |
| [运动后] [服药后] [不适]                         |
+--------------------------------------------------+
| 备注（可选）                                     |
| [______________________________]                 |
+--------------------------------------------------+
|              [ 保存记录 ]                        |
+--------------------------------------------------+
```

### 4.4 关键交互

- 默认填充当前时间
- 输入完成后点击保存
- 保存成功后返回首页并刷新趋势图

### 4.5 校验规则

- 收缩压必填
- 舒张压必填
- 心率必填
- 输入必须为合理数字范围

## 5. 编辑记录页

编辑记录页结构与新增记录页一致。

差异：

- 标题为“编辑记录”

- 保存按钮文案改为“保存修改”

## 7. 历史记录页

### 7.1 页面目标

让用户快速浏览和管理过去记录。

### 7.2 页面结构

1. 顶部导航栏
- 标题：历史记录
- 可选报告快捷入口

2. 时间范围筛选区，可选
- 最近 7 天
- 最近 14 天
- 最近 30 天

3. 记录列表
- 每条显示时间、收缩压/舒张压、心率、直接状态说明
- 点击进入详情

### 7.3 页面草图

```text
+--------------------------------------------------+
| 历史记录                                   报告  |
+--------------------------------------------------+
| [7天] [14天] [30天]                              |
+--------------------------------------------------+
| 03/29 08:30   128/82   76   正常范围内           |
| 03/28 20:10   142/90   79   收缩压和舒张压偏高   |
| 03/28 08:15   130/84   74   收缩压偏高           |

```

- 无数据时显示空状态说明

- 舒张压曲线

5. 记录点详情预览区域，可选
- 点击某个点后展示对应记录的直接解释

## 13. 空状态与异常状态

### 13.1 首页空状态

- 显示简短引导说明
- 主按钮“添加第一条记录”
- 隐藏趋势图实际数据，保留占位说明

### 13.2 历史页空状态

- 文案提示暂无记录
- 提供新增记录入口

### 13.3 提醒页空状态

- 提示开启提醒帮助坚持测量
- 提供新增提醒入口

### 13.4 趋势页异常状态

- 异常点高亮
- 摘要直接写明是收缩压偏高、舒张压偏高或两者偏高
